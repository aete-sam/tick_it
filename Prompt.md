# TickIt Project Handoff & Context Document

This document compiles the complete state of the **TickIt** task-management Flutter application.

---

## 1. Project Directory & File Guide

Below is the directory structure of the `lib` folder and the purpose of every file:

```text
lib/
├── main.dart                      # App entry point (initializes Firebase & system overlay preferences)
├── app.dart                       # Root widget (configures MaterialApp, MultiProvider, Routing, and Theme)
├── firebase_options.dart          # Native platform configuration file auto-generated by FlutterFire CLI
│
├── config/                        # Configuration files
│   ├── routes.dart                # named routing definition (AppRoutes) and route transition builder (AppRouter)
│   └── theme.dart                 # App design system, text styles (Google Fonts), color palettes (AppColors)
│
├── models/                        # Data models
│   └── task_model.dart            # TaskModel structure with fields for ID, Title, Category, DateTime, Start/End times, Priority, Description, isCompleted, and userId
│
├── providers/                     # App state management (ChangeNotifier Providers)
│   ├── auth_provider.dart         # Exposes authentication states, display names, loading indicators, and wraps AuthService
│   └── task_provider.dart         # Exposes tasks, allTasks, categoryCounts, and CRUD operations wrapping TaskService
│
├── services/                      # Low-level service layers (Firebase integration)
│   ├── auth_service.dart          # Interacts with Firebase Auth & Google Sign-In, caches names to SharedPreferences
│   └── task_service.dart          # Handles Firestore CRUD operations and caches updates locally to SharedPreferences
│
├── screens/                       # User Interface Screens
│   ├── splash_screen.dart         # Animated onboarding screen (Refactored to Provider)
│   ├── login_screen.dart          # Plain white auth entry with Google/Email login options (Refactored to Provider)
│   ├── signup_screen.dart         # User registration screen (Uses direct setState/AuthService; next for refactoring)
│   ├── home_screen.dart           # Dashboard with weekly calendar, task category stats, and sorted task list
│   ├── create_task_screen.dart    # Full-screen form to add tasks (with priority levels: Low, Medium, High)
│   └── edit_task_screen.dart      # Task edit screen with an integrated deletion option
│
└── widgets/                       # Reusable UI components
    ├── category_card.dart         # Renders card layout for categories (Project, Work, etc.)
    ├── custom_text_field.dart     # General styled form input field
    ├── google_sign_in_button.dart # Custom button utilizing flutter_svg to load a Google logo
    ├── gradient_background.dart   # Decorative backdrop painted with custom shapes/curves
    └── task_tile.dart             # Indicated outline task list card (thicker 2.2px borders with priority coloring)
```

## 2. Complete Context & Handoff Prompt for AI Agents

Copy and paste the prompt block below to instantly bring any developer AI agent up to speed on the project:

```markdown
You are an expert Flutter developer assisting with the "TickIt" task-management app. 
Here is a complete summary of the project architecture and the active tasks to avoid redundant file analysis.

### Tech Stack & Dependencies
- **Core:** Flutter, Dart
- **Backend:** Firebase (Firebase Auth, Cloud Firestore)
- **Local Cache:** SharedPreferences (caches display name and task database for offline fallback)
- **UI Packages:** google_fonts (Poppins), table_calendar, flutter_svg (used for loading vectors like google.svg)
- **State Management:** Provider (MultiProvider wrapping MaterialApp in lib/app.dart)

### Architecture Patterns
- **Services (services/):** Direct SDK API layers interfacing with Firebase.
- **Providers (providers/):** Wraps services in ChangeNotifiers. They expose isLoading, error messages, and state fields, and trigger notifyListeners().
- **Screens (screens/):** Build the UI. Instead of instantiating services, they watch state using `context.watch<T>()` and trigger logic actions using `context.read<T>()`.
- **Styling (widgets/ / theme.dart):** Visual layouts use custom borders corresponding to priority (High: red, Medium: orange, Low: yellow, Completed: green).

### Current Codebase Implementations
1. **Splash Screen:** Refactored to Provider. Checks `authProvider.isLoggedIn` and auto-navigates.
2. **Login Screen:** Refactored to Provider. Has a plain white background, updated font formatting, and uses `SvgPicture` for the Google sign-in button.
3. **Task Cards:** Clicking anywhere on the card opens the Edit Task screen (chevron icon shown on right). Checkbox triggers toggle completion. Outlines are thick (2.2px) and priority-colored.
4. **Task Dashboard Sorting:** Tasks are sorted on the home screen by priority (High -> Medium -> Low).
5. **Dynamic Categories:** Home screen displays only the category cards containing tasks.

### Next Tasks to Implement
1. **Signup Screen (`lib/screens/signup_screen.dart`):** Refactor from direct `AuthService` state to `AuthProvider`.
2. **Home Screen (`lib/screens/home_screen.dart`):** Refactor to use `TaskProvider` for listening to tasks and handling updates.
3. **Create & Edit Screens (`lib/screens/create_task_screen.dart`, `lib/screens/edit_task_screen.dart`):** Refactor to use `TaskProvider` methods to save/delete/update tasks.
```
